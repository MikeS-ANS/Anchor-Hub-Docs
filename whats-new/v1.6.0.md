# v1.6.0 — Company Directory v2, Multi-Service Mappings & Bug Fixes

## Company Directory Improvements

The Hub Company Directory has been significantly expanded with new editing capabilities and AT data enrichment.

### Inline Platform Chip Editing
Platform name chips (Kaseya, Blackpoint, Pax8) are now fully editable without leaving the Companies tab:
- **×** on any chip to remove that platform mapping
- **Click a chip** to reassign it to a different AT company (searches your hub companies in memory)
- **+** on an empty cell to add a new Kaseya or Blackpoint name manually

### AT Classification
Each company now displays its Autotask Classification label (e.g. TC, TC+, TC-Lite, Onboarding, Co Managed) as a small chip beneath the company name. Use the **↻ Update Classifications** button to pull the latest values from Autotask.

### Excluded Companies Toggle
Excluded companies are hidden by default. A **Show excluded** checkbox in the Companies tab header reveals them when needed.

### Pax8 Sync Tab
Pax8 Sync controls and log have moved into the dedicated Pax8 Sync tab, keeping the Companies tab clean.

---

## Service Mappings: Multi-Service Support

Vendor products in Service Mappings can now map to **multiple AT services** — useful for Blackpoint Security Plus which bills against two different service line IDs depending on the contract.

### What changed in the edit UI
- **Vendor product name** is now editable as a text field
- **AT Services** are listed as individual chips — click × to remove one, or search to add another
- **Contracts** field is editable as comma-separated text
- Changes are saved all at once when you click **Save**

### Blackpoint Security Plus
The legacy `sp-primary` / `sp-alt` split has been merged into a single `Security Plus` entry with both AT service IDs (119 and 263). The system automatically picks the most common one for new pushes.

## Bug Fixes

- **Hub corruption recovery** — if the hub file was accidentally overwritten with a Blackpoint compat format, it is now auto-detected and repaired on next load
- **Pax8 column blank after sync** — fixed; was treating the Pax8 array as a single object
- **"Not yet synced" status after sync** — fixed; version check was rejecting recovered v3 hubs
- **Transparent service mapping search dropdown** — fixed CSS variable reference
- **BlackPoint Invoice Processor crash on first load** — fixed null reference when hub is v3 format
- **BlackPoint Invoice Processor** — removed internal SharePoint path hint, renamed button to "Process Invoice", renamed tool to "BlackPoint Invoice Processor"

The tools list now scrolls independently. Home and Apps remain pinned at the top; Help and Settings remain pinned at the bottom.
