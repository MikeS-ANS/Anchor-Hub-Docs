# v1.5.2 — Blackpoint Billing Automation

**Released:** June 2026

## What's New

### Blackpoint / CompassOne — Full Billing Automation

The Blackpoint tool has been completely redesigned. Instead of pulling device counts live from the Blackpoint API, it now reads directly from the **Account Usage Report CSV** that Blackpoint exports monthly to SharePoint — giving you accurate billing figures with no API rate limits, no delays, and a full side-by-side comparison against what's currently billed in Autotask.

**New Compare & Push tab:**

- **Pick a file** — select a year and CSV file from SharePoint; the tool loads it in seconds
- **Comparison table** — every Blackpoint company side-by-side with its Autotask contract: current billed units, what Blackpoint says it should be, and the delta
- **Push changes** — push Security+ and 365 Defense adjustments to Autotask ContractServiceAdjustments in one click, with a confirmation dialog showing exactly what will change before anything fires
- **Company matching** — Blackpoint company names are automatically fuzzy-matched to Autotask. High-confidence matches are auto-confirmed and saved to SharePoint so every user gets the same matches. Low-confidence matches get a "Fix" button to set them manually — one time, shared with everyone.
- **Clickable filter bar** — click any stat (matched, need match, AT errors, SP changes, 365D changes, excluded) to filter the table to just those rows
- **Exclude companies** — click ✕ next to any company to exclude it from billing; ↩ to bring it back. Saved to SharePoint.
- **Export to Excel** — export the full comparison table with color-coded delta columns

**365 Defense billing logic:** Security+ includes `floor(MDR × 1.3)` Cloud Response seats at no charge. The table shows CR Included, CR Used, and CR Billable separately so you can see exactly how the billable count is calculated.

**Auto-create missing services:** If a company's Autotask contract is missing the Security+ or 365 Defense service line, the tool creates it automatically from the catalog price rather than skipping the company.

**Push confirmation dialog:** Before any Autotask write happens, a table shows every company that will be updated — current units, new units, and the change. Cancel at any point to abort.

**Push audit log:** Every push is logged to `ANS-Vendors/Blackpoint/push-log.json` on SharePoint with the date, user, service type, and per-company results.

### Polish & Performance

- **30-minute AT data cache** — the bulk Autotask data load runs once per session; subsequent single-row re-runs (after confirming a company match) reuse the cache, making them near-instant
- **Security+ service ID auto-detection** — the tool scans existing contracts to determine whether your Autotask setup uses service ID 119 or 263 for Security+, and uses the correct one automatically
- **↺ Retry errors button** — if any rows fail with an AT error, a retry button appears in the filter bar to re-run just those rows without reloading everything
- **↻ Refresh file list button** — re-fetches the SharePoint file list without leaving the tool

## How to Update

The app updates automatically. If you haven't seen the update prompt yet, restart Anchor Hub.
