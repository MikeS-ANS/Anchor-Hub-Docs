# v1.5.3–v1.5.6 — Role-Based Access & Permissions

**Released:** June 2026

## What's New

### Role-based sidebar filtering (v1.5.4)

Anchor Hub now filters the sidebar based on the signed-in user's Azure AD App Role. Each person only sees the tools their role allows — no configuration required on the user's end.

Roles are assigned automatically via **Azure AD Dynamic Groups** based on job title. When someone's title changes in M365, their Hub access updates within minutes.

**Nine Hub roles are defined:**

| Role | Tools |
|---|---|
| `hub.admin` | All tools |
| `hub.manager` | M365 Subscription Comparison, Project Time Summary, Duo Management, Project Profitability |
| `hub.delivery` | Duo Management |
| `hub.tam` | Duo Management |
| `hub.strategic` | M365 Subscription Comparison, BlackPoint Usage, MSC Agreements, Duo Management |
| `hub.projects` | Project Time Summary, Project Profitability |
| `hub.finance` | M365 Subscription Comparison, Pax8 Invoice Processor, M365 Margin Analyzer, Pax8 Invoice Comparison, Kaseya Invoice Processor, Autotask Contract Changes, Autotask Contract Renewals, BlackPoint Usage, MSC Agreements |
| `hub.sales` | None currently |
| `hub.wsd` | None currently |

If a user has no role assigned and no individual overrides, all tools are shown as a fallback.

### Per-user tool overrides (v1.5.5)

You can now grant a specific person access to an individual tool without assigning them a full role. This is managed through the **Hub User Overrides** SharePoint list — one row per person per tool.

**Example:** To give Pat access to BlackPoint Usage without giving him the full `hub.finance` role:
1. Open **SharePoint Intranet → Lists → Hub User Overrides**
2. Add a row: Title = `patm@anchornetworksolutions.com`, ToolKey = `blackpoint-processor`
3. Pat restarts Anchor Hub — the tool appears in his sidebar

Overrides are unioned on top of role-based access, so a user's allowed tools are the combination of both.

See [Roles & Permissions](roles-and-permissions.md) for full instructions and the complete tool key reference.

### Hatz.ai AI engine (v1.5.3)

The AI analysis feature in Pax8 Invoice Comparison now runs through the **Hatz.ai** gateway instead of direct Anthropic API calls. The API key is managed centrally in Azure Key Vault — there is no longer an AI key field in Settings. No action needed from users.

### Roles & Permissions help page (v1.5.6)

The in-app Help page now includes a full **Roles & Permissions** section covering:
- How the role system works
- Role-to-tool access table
- Step-by-step instructions for updating the Hub Role Matrix
- How to add a new tool to the matrix when one is deployed
- How to use Hub User Overrides for individual grants
- Full tool key reference

The Tools Overview section was also updated to document all 13 tools (previously only 9 were listed).

## How to Update

The app updates automatically. If you haven't seen the update prompt yet, restart Anchor Hub.
