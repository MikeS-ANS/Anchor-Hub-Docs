# v1.4.4 — Project Profitability Improvements

**Released:** June 2026

## What's New

### Project Profitability

A round of accuracy and usability fixes based on real-world testing.

**Data accuracy**
- **Hours** now pull from all time entries on the contract — tickets and project tasks combined. Previously only project task hours were counted, which caused hours to show lower than Autotask's contract view.
- **Invoiced** now correctly shows milestones in **Billed** status only.
- **Pending** now captures milestones in both **In Progress** and **Ready to Bill** status.
- **Gross Margin %** and **Effective Rate** are now calculated against invoiced + pending combined, giving a more accurate picture of total contract value.

**Single project view**
- The detail card now shows all columns: Est Hours, Billed Hours, Total Hours, Hours Variance %, Effective Rate, and Gross Margin $.

**Look up by project number**
- Type a project number directly (e.g. `P20250826.0001`) to pull a single project without setting a date range.

**Other improvements**
- Date range defaults to first of the current month instead of two years ago.
- Filter box to narrow results by project number when running a date range report.
- Removed the **vs Rack** column — it wasn't adding actionable information.

## How to Update

The app updates automatically. If you haven't seen the update prompt yet, restart Anchor Hub.
