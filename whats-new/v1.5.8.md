# v1.5.8 — Kaseya Invoice Processor: Bundled Billing

## Workplace Bundle Overrides

Clients with contracted bundled Workplace/DFP seats can now be configured in **Kaseya → Settings → Workplace Bundle Overrides**.

- Add a row for each client/product combination with the number of contracted bundled seats
- Product options: all six DWP and DFP license types (Metered User, Metered Server, Unlimited User, Unlimited Server, DFP Laptop/Desktop, DFP Server)
- Bundled seats are automatically **deducted from the billable count** in the Workplace Usage table before pushing to Autotask
- The **QBO Bundled entry uses the actual invoice avg rate × bundled seats** for that product — no more manual percentage splits

---

## SaaS Bundled Billing

A new **Bundled?** column appears in the Datto SaaS Usage table for each invoice you process.

**Checking a client as bundled:**
- Toggle the checkbox for any client — the flag saves immediately to the hub directory (persists across months)
- Once checked, the tool looks up that client's included seats from the **Managed Service Client MSC.xlsx** file in SharePoint
- The table shows **Bundled Qty** (from SP) and **Billable** (Invoice Qty − Bundled Qty) as separate columns
- QBO journal entries automatically split into **Datto SaaS – Standalone** and **Datto SaaS – Bundled** with actual dollar amounts based on seat counts

**Manual override:**
- If the SP file value isn't right for a given month, click the **✎ pencil** next to any client's bundled qty to override it
- Overridden values appear in **amber** so it's immediately obvious a manual value is in use
- Click **✕** to clear the override and return to the SP file value
- Overrides persist in the hub directory and apply to both the QBO entries and the Autotask push

## QBO Journal Entries

- **Reconcile flags removed** — the yellow row highlights and ⚠ reconcile badges are gone
- **Sortable columns** — click any column header (Description, QBO Account, Class, Amount) to sort; click again to reverse

## Bug Fixes

- Fixed a crash that prevented invoice results from rendering (`fmtAmt is not defined`)
- Fixed client name matching for companies with spacing differences (e.g. "39North LLC" now correctly matches "39 North LLC" in the SP file)
- Revenue SP file lookup switched to direct path — the previous DocId-based search was not reliably indexed
- Removed the old **Bundled Splits %** settings for SaaS and DWP — these are fully replaced by the seat-based calculations above
