# v1.5.8 — Kaseya Invoice Processor: Full Rebuild + Bundled Billing

## Kaseya Invoice Processor — Full Rebuild

The Kaseya Invoice Processor has been completely rebuilt from the ground up. The old version required downloading invoice files manually and stored data locally. The new version is fully SharePoint-integrated, has a redesigned UI, and handles the full billing workflow end-to-end.

### What's new in the rebuild

**SharePoint-native invoice loading**
- Browse invoice files directly from SharePoint (ANS-Vendors → Kaseya/Invoices) — no manual downloads
- Year folders with inline grand totals from prior runs so you can see history at a glance
- Invoice date extracted automatically from the filename

**Redesigned results view**
- Metric cards at the top (Invoice Total, Invoice Date, Client Count)
- Nine data tables: QBO Journal Entries, Totals by Module, Totals by Category, Anchor Tools, Datto Workplace Costs, Datto Workplace Usage, Datto SaaS Usage, Datto Backup & Networking, Totals by Client
- Step-by-step flow: select → process → review → export/push

**Smart column detection**
- Automatically finds the right columns (Company, Module, Qty, Rate, Total, etc.) even when Kaseya changes their layout
- Fallback rules for rows with blank modules (ConnectBooster, Shipping, Bitdefender)
- Parsing warnings surface any rows that couldn't be classified

**QBO Journal Entries**
- Auto-generated lines for every product with correct QuickBooks account paths and class splits
- PSA, RMM, and IT Glue costs split across Strategic Services / Service Delivery / Admin based on configurable percentages
- Sortable by any column

**Autotask Push**
- Push Datto Workplace seat counts to Autotask contract service lines — creates adjustments effective the first of the month
- Push Datto SaaS license counts (service IDs 98 and 88)
- Preview modal before committing any changes
- Real-time progress streaming: per-company and per-product status (updated, no change, skipped, error, no contract)
- Automatic contract rollover handling

**Snapshot & delta comparison**
- Auto-saves each processed invoice as a monthly snapshot
- Compare any two SharePoint invoices or saved snapshots side-by-side
- Delta report shows grand total change, module-level changes, and per-client breakdowns — only changed items shown

**Organization mapping**
- Auto-matches Kaseya company names to Autotask companies using token similarity
- Manual mapping with live Autotask search for non-matching names
- All mappings stored in the centralized hub directory (shared with other tools)
- Bulk "Accept All High Confidence" for one-click confirmation

**Excel export**
- Multi-sheet formatted workbook with all nine data tables
- Separate QBO Entries sheet ready to copy into QuickBooks

## Bundled Billing (new in v1.5.8)

### Workplace Bundle Overrides
- New Settings section: add per-client contracted bundled seat overrides by product type
- Six products supported: all DWP and DFP license types
- Bundled seats deducted from billable count before AT push
- QBO bundled amount = bundled seats × actual avg rate from the invoice (not a percentage)

### SaaS Bundled Billing
- **Bundled?** checkbox column in the Datto SaaS Usage table — one toggle per client, saved permanently to hub directory
- Bundled qty loaded from **Managed Service Client MSC.xlsx** in SharePoint (ANS-Finance → Managed Service Clients)
- Billable qty = Invoice qty − Bundled qty, shown as a separate column
- QBO entries automatically split into Standalone and Bundled lines with actual dollar amounts
- **Manual qty override** — ✎ pencil to set a custom bundled qty per client; shown in amber when active; ✕ to clear
- Overrides persist in the hub directory and apply to QBO entries and the AT push

### Other improvements in v1.5.8
- Removed yellow row highlights and ⚠ reconcile badges from QBO table
- QBO column headers are now sortable
- Fixed client name matching for spacing differences (e.g. "39North LLC" vs "39 North LLC")
- Fixed a crash that prevented invoice results from rendering after the bundled billing refactor
